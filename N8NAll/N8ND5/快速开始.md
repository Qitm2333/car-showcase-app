# 🚀 快速开始指南

## 只需3步，立即开始使用！

---

### 📝 步骤 1：导入n8n工作流

1. 打开你的n8n实例
2. 点击 **"导入工作流"**
3. 选择文件：`整合工作流-输入车型获取图片.json`
4. 确保Google Sheets凭证已配置
5. **激活工作流**
6. 复制Webhook URL（例如：`https://你的域名/webhook/get-car-images`）

---

### 🎨 步骤 2：打开页面并配置

1. 双击打开 **`汽车图片查询系统.html`**

2. **点击右下角的⚙️按钮**

3. 在弹出的调试面板中：
   ```
   🔗 Webhook配置
   ├─ 环境选择：选择"生产环境"
   ├─ 生产环境URL：粘贴你的webhook地址
   └─ 请求超时：30000（30秒）
   ```

4. 点击 **"测试连接"** 按钮
   - ✅ 如果显示"连接成功"，进入下一步
   - ❌ 如果失败，检查URL是否正确、n8n是否运行

5. 点击 **"保存配置"** 按钮

---

### 🔍 步骤 3：开始搜索！

1. 在搜索框输入车型名称，例如：
   - 奥迪A6L
   - 宝马X5
   - 特斯拉Model 3

2. 点击 **"🚀 搜索图片"** 按钮

3. 等待几秒，查看结果！

---

## 💡 没有n8n服务器？试试模拟模式！

如果你还没有n8n服务器，可以使用**模拟数据模式**测试前端：

1. 点击右下角⚙️打开调试面板
2. 打开 **"使用模拟数据"** 开关
3. 点击"保存配置"
4. 现在可以测试前端功能了（不需要真实的n8n服务器）

## ❓ 常见问题

### Q1: 点击搜索后一直转圈圈？

**解决方法：**
1. 检查webhook URL是否正确
2. 确认n8n工作流已激活
3. 在调试面板点击"测试连接"查看具体错误

### Q2: 显示CORS错误？

**解决方法：**

在n8n的"返回响应"节点中添加Response Headers：
```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: POST, OPTIONS
Access-Control-Allow-Headers: Content-Type
```

### Q3: 连接超时？

**解决方法：**
1. 打开调试面板
2. 将"请求超时时间"改为60000（60秒）
3. 点击"保存配置"

### Q4: 找不到车型？

**可能原因：**
- 车型不在Google Sheet数据库中
- 车型名称拼写有误

**解决方法：**
- 尝试不同的车型名称
- 查看浏览器Console的日志（按F12）
